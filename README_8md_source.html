3 The program has been developed on an Arduino Mega 2560 with an Ethernet shield
4 from the first generation (with W5100 chip). It is believed, that, possibly after

8 ## Software architecture
9 There is a few classes, (GlobalCovfefe and a few derived classes)
10 contained in the `src` directory. These classes operates
11 on a `Stream` (in the Arduino sense). This is typically, but not necessarily,
12 an `EthernetClient` instance.
13 
14 The "example" `GlobalCovfefe.ino` contains the "application". It controls the
15 Ethernet connections, and delivers it, as a `Stream` to the class `GlobalCovfefe`.
16 
17 Most important difference from the GlobalCache devices
18 is that the current program is single-threaded.
19 That is, it accepts one command, executes it, and only when it is finished
20 (or times out), the program listens for new commands.
21 Thereby, commands like `stopir` are not meaningful, and not implemented.
22 
23 For this reason, only one connection at a time is possible.

30 ## Configuration
31 The GlobalCache hardware is extensively configurable, both using the TCP commands,
32 as well as using the built-in HTTP-server. This program is instead configured
33 with CPP defines in the file `config.h`.

45 ## Porting
46 GlobalCovfefe as such is a portable program, not a program that needs porting.
47 There are no other "dependencies" than "at least *X* of *Y*".
48 However, Infrared4Arduino only runs
49 on some hardware. Porting it to more hardware platforms is desirable and planned.
